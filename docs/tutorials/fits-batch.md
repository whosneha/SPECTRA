# Tutorial: FITS Batch Processing

Learn how to process multiple objects from FITS catalogs efficiently.

## Overview

This tutorial demonstrates processing cluster photometry from FITS catalogs generated by LSST or similar surveys.

## Prerequisites

- FITS catalogs with photometry data
- Configured `config.yaml`
- SPECTRA installation

## Step 1: Prepare Your Data

Organize your FITS files in a directory:

```
/data/catalogs/
├── cluster_catalog_1.fits
├── cluster_catalog_2.fits
└── cluster_catalog_3.fits
```

## Step 2: Configure Batch Processing

Edit `config.yaml`:

```yaml
input:
  type: 'fits_batch'
  fits_dir: '/data/catalogs'
  file_pattern: 'cluster_catalog_*.fits'
  max_rows_per_file: 20  # Process first 20 clusters per file

fitting:
  method: 'mcmc'
  parameters: ["mass", "age", "metallicity", "dust"]
  priors:
    mass: [8.0, 13.0]
    age: [0.001, 13.0]
    metallicity: [-2.0, 0.5]
    dust: [0.0, 2.0]

mcmc:
  n_walkers: 32
  n_steps: 2000
  n_burn: 500

plotting:
  output_dir: './output/batch_run'
```

## Step 3: Run Batch Processing

```bash
python src/main.py
```

## Advanced: Custom Row Selection

Process specific interesting objects:

```yaml
input:
  type: 'fits_batch'
  fits_dir: '/data/catalogs'
  file_pattern: 'cluster_catalog_1.fits'
  row_indices: [0, 5, 12, 23, 45]  # Specific clusters
```

## Tips

1. **Start small**: Test with `max_rows_per_file: 5` first
2. **Check failures**: Review error messages for problematic rows
3. **Save MCMC chains**: Use HDF5 backend for later analysis
4. **Parallel processing**: Future feature for faster processing
